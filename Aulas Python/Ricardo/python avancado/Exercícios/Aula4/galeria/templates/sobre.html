Sobre o Flask

A Decorator @app.route

O decorator @app.route em Flask serve para mapear uma rota HTTP (por padrão GET) a uma função Python dentro da aplicação. É assim que associamos URLs a comportamentos específicos no servidor.

O que são Rotas?

Rotas são caminhos ou URLs que identificam recursos ou funcionalidades na aplicação Flask. Por exemplo, a rota /sobre invoca a função que renderiza a página "Sobre". 

@app.route('/sobre')
def sobre():
    return render_template('sobre.html')

Neste exemplo, ao aceder http://localhost:5000/sobre, a função sobre() será executada e retornará o template correspondente.

O Protocolo HTTP

HTTP (Hypertext Transfer Protocol) é o protocolo que define a comunicação entre clientes (navegadores, apps) e servidores web. Ele regula como as solicitações e respostas devem ser formatadas e transmitidas.

O Método GET

O método GET é usado para recuperar dados sem alterar o estado do servidor. Em Flask, rotas sem métodos explícitos aceitam GET por padrão, sendo ideal para páginas estáticas ou consultas simples.

Endpoints

Um endpoint combina uma URL e um método HTTP para definir um ponto de acesso a um recurso ou funcionalidade. No Flask, cada rota registrada com @app.route cria um endpoint.

Exemplo de endpoint para listar itens: @app.route('/itens') com método GET.

Resumo

Rotas: URLs que acionam funções Python na app.

HTTP: Protocolo de comunicação web.

GET: Método usado para obter dados.

Endpoints: Combinação de rota e método HTTP que expõe funcionalidades.
